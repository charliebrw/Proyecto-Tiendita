51       /**
52        * The `offset` modifier can shift your popper on both its axis.
53        *
54        * It accepts the following units:
55        * - `px` or unit-less, interpreted as pixels
56        * - `%` or `%r`, percentage relative to the length of the reference element
57        * - `%p`, percentage relative to the length of the popper element
58        * - `vw`, CSS viewport width unit
59        * - `vh`, CSS viewport height unit
60        *
61        * For length is intended the main axis relative to the placement of the popper.<br />
62        * This means that if the placement is `top` or `bottom`, the length will be the
63        * `width`. In case of `left` or `right`, it will be the `height`.
64        *
65        * You can provide a single value (as `Number` or `String`), or a pair of values
66        * as `String` divided by a comma or one (or more) white spaces.<br />
67        * The latter is a deprecated method because it leads to confusion and will be
68        * removed in v2.<br />
69        * Additionally, it accepts additions and subtractions between different units.
70        * Note that multiplications and divisions aren't supported.
71        *
72        * Valid examples are:
73        * ```
74        * 10
75        * '10%'
76        * '10, 10'
77        * '10%, 10'
78        * '10 + 10%'
79        * '10 - 5vh + 3%'
80        * '-10px + 5vh, 5px - 6%'
81        * ```
82        * > **NB**: If you desire to apply offsets to your poppers in a way that may make them overlap
83        * > with their reference element, unfortunately, you will have to disable the `flip` modifier.
84        * > You can read more on this at this [issue](https://github.com/FezVrasta/popper.js/issues/373).
85        *
86        * @memberof modifiers
87        * @inner
88        */
