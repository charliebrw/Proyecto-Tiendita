48                 be returned to the server on all HTTP and HTTPS requests. This is the default value because it ensures

164                 The LoginPath property informs the middleware that it should change an outgoing 401 Unauthorized status
165                 code into a 302 redirection onto the given login path. The current url which generated the 401 is added
166                 to the LoginPath as a query string parameter named by the ReturnUrlParameter. Once a request to the
167                 LoginPath grants a new SignIn identity, the ReturnUrlParameter value is used to redirect the browser back  
168                 to the url which caused the original unauthorized status code.
169                 
170                 If the LoginPath is null or empty, the middleware will not look for 401 Unauthorized status codes, and it will
171                 not redirect automatically when a login occurs.
172

174             
175                 
176                 If the LogoutPath is provided the middleware then a request to that path will redirect based on the ReturnUrlParameter.
177                 
178             
179             
180                 
181                 The ReturnUrlParameter determines the name of the query string parameter which is appended by the middleware
182                 when a 401 Unauthorized status code is changed to a 302 redirect onto the login path. This is also the query 
183                 string parameter looked for when a request arrives on the login path or logout path, in order to return to the 
184                 original url after the action is performed.
